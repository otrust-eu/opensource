Service Description

OTRUST provides four main services:

  • Timestamp: Blockchain timestamping using OpenTimestamps. We create cryptographic records that data existed at a specific point in time by anchoring SHA-256 hashes to the Bitcoin blockchain.
  • ID: Commitment-based ID packages. Verify facts such as age or unique identity without exposing underlying documents to verifiers.
  • Sign: Multi-party document signing with blockchain evidence. Signers verify document hashes and sign cryptographically, with the final signature package anchored to Bitcoin.
  • Auth: Hosted Auth for partners. Short-lived challenges, branded login screens, and callback tokens bound to state and registered redirect URIs.

OTRUST is free to use. No account required. No payment needed.

How Timestamps Work

Submission: When you submit a hash, we create a timestamp claim and submit it to OpenTimestamps calendar servers for Bitcoin anchoring.

Confirmation: OpenTimestamps batches anchors to Bitcoin roughly every hour. Full Bitcoin confirmation can take longer; until then the anchor is pending but the receipt can still be inspected.

Verification: Completed receipts can be verified independently using any OpenTimestamps-compatible client. You do not need to trust OTRUST for verification.

How Sign Works

Document Handling: You can choose between temporary upload (encrypted, auto-deleted within 1-12 hours) or local-hash mode where you share the document directly with signers.

Signing Process: Each party receives a unique link to verify the document hash and sign cryptographically. Signatures are collected and the final package is anchored to Bitcoin.

Data Retention: For temporary uploads, documents are encrypted at rest and automatically purged. Signature records (hashes and metadata) are permanent.

Legal note: OTRUST Sign provides cryptographic evidence of signature intent. Signatures are Ed25519 attestations anchored to Bitcoin, showing that parties reviewed and acknowledged a document hash at a specific time. Legal standing varies by jurisdiction and context.

How ID Works

Identity Verification: You upload an ID document and take a selfie for liveness detection. Sensitive checks run in your browser before the ID package is produced.

Commitment ID packages: The service stores a commitment-bound ID package and verification metadata, not your raw passport image or selfie.

Sybil resistance: For unique identity IDs, a one-way hash of your ID number can limit one ID per person while keeping the actual ID private.

Data handling: Only the ID package and public metadata needed for verification are stored server-side.

Privacy note: Verifiers learn that the claim is satisfied (for example "18 or older"), not your underlying document fields.

How Auth Works

Challenges: Partners create short-lived authentication challenges through the API with a registered redirect URI and state parameter.

Hosted flow: Users complete ID verification on otrust.eu. Partner branding is limited to approved visual elements with visible OTRUST disclosure.

Callbacks: Your server must verify the returned token, challenge binding, state, and expiry before establishing a session.
