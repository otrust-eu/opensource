Operating Model

OTRUST focuses on local-first cryptographic workflows where the service boundary is explicit and each verification artifact can be checked later without opaque platform assumptions.

Design Direction

OTRUST is built to minimize raw data handling while still producing records that remain useful for later verification. The focus is practical cryptographic evidence, transparent boundaries, and open-source implementation paths.

Use Cases

  • Timestamped evidence for intellectual property and research artifacts
  • Hash-based document signing workflows for agreements
  • ID and eligibility checks with selective disclosure
  • Partner login flows where OTRUST remains the visible verification domain

Created By

OTRUST was created by Kris Ledel (Kristian Ledel), who designed and built the platform architecture, security model, APIs, and user flows.

Open Source
