About OTRUST

OTRUST provides cryptographic trust infrastructure for proving when records existed, who signed them, and whether a user controls an approved identity proof in partner login flows across global digital identity markets.

What is OTRUST?

OTRUST provides four connected trust services:

  • Timestamp: Creates cryptographic proof of when digital content existed. Using OpenTimestamps, proofs are anchored to the Bitcoin blockchain and can be verified independently, forever.
  • Signed: Multi-party document signing with blockchain proof. Prove WHO signed WHAT and WHEN, with the final signature package anchored to Bitcoin.
  • Proof: Privacy-preserving proof packages using commitments. Prove facts about yourself without storing the raw identity data behind the claim.
  • Sign in: Hosted identity login where partner applications can verify proof ownership through short-lived challenges, approved branding, and global eID-ready flows.

All services are built around the same principle: OTRUST should see as little raw content as possible while still producing records that can be verified later.

Customer readiness: Partner-branded auth supports both hosted login and proof creation flows, including client configuration, audit logging, and visible OTRUST disclosure.

  • Global digital identity architecture - plan identity-provider-neutral login flows without hiding the role of each provider
  • KYC-free verification – pass identity checks without exposing sensitive information

How Verification Works

Anyone with your original file and the .ots proof can verify the timestamp. Verification checks the Merkle path from your file's hash to the block header. No trust in OTRUST required – the math proves itself.
