Hosted flow

Hemsted sessions verify with OTRUST ID.

When Hemsted starts an Auth session, OTRUST presents a hosted ID-verification screen tied to that challenge. Hemsted branding provides context, and OTRUST disclosure makes it clear that verification happens on otrust.eu.

Step 01

Challenge starts at Hemsted

Hemsted creates the short-lived auth challenge and sends the user to the hosted OTRUST login URL.

Step 02

ID verification on OTRUST

The user verifies control of an OTRUST ID on otrust.eu with Hemsted-approved visual context.

Step 03

Return after verification

OTRUST redirects back to Hemsted's registered callback with the expected token and state handling.

Production controls

Theme configuration for Hemsted's hosted Auth flow.

The hosted theme controls brand presentation only. It does not change challenge creation, callback handling, token verification, or registered redirect behavior.

Live flow

Challenge-bound verification, unchanged callback.

Hemsted creates the challenge server-side and redirects to the OTRUST hosted Auth URL. After ID verification, OTRUST returns to Hemsted's registered callback with the same token/code and state behavior as the standard hosted flow.

01

Partner creates challenge

Hemsted starts the session and receives an OTRUST hosted login URL tied to that short-lived challenge.

02

User verifies ID

The user completes OTRUST ID verification on otrust.eu with Hemsted branding and OTRUST disclosure visible.

03

Callback returns to Hemsted

The callback remains registered, scoped, and state-bound. Partner branding does not change auth semantics.
