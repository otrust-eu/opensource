Developer integration

Add an OTRUST Auth button by creating a challenge server-side, redirecting to the returned Auth URL, then verifying the callback token before creating a session.

Create a challenge

import { auth } from '@otrust/sdk'

const state = auth.generateState()
const challenge = await auth.createChallenge({
  clientId: 'my-app',
  redirectUri: 'https://app.example/auth/callback',
  scope: ['identity'],
  state
})

if (challenge.ok) {
  res.redirect(challenge.value.loginUrl)
}

Verify the callback

const token = req.query.token
const result = await auth.verify(token)

if (result.ok && result.value.valid) {
  const proofId = result.value.proofId
  const verified = result.value.identity?.verified
}
Direct visits to /auth/login are intentionally not useful. That URL needs a short-lived challenge from your application.

Brand-guide checklist

  • Logo must be served over HTTPS and stay below 200KB.
  • Colors are validated as hex, rgb, or rgba values.
  • Copy is rendered as text only, never as injected HTML.
  • The same theme follows users to /proof?auth_challenge=... if they need to create an ID first.
  • Every change is written to the OTRUST audit log with client_id, actor, fields, and timestamp.
